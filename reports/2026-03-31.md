### 3. NousResearch/hermes-agent — 自进化 Agent 框架（今日 ⭐+1,851，总 ⭐18,552）

> **仓库**：https://github.com/NousResearch/hermes-agent
> **定位**：可自我改进的长期 AI Agent 运行平台

**这是什么：** Nous Research 开源的 Agent 框架，核心差异点是**闭合学习循环**——Agent 会从完成复杂任务后自动创建新 Skill、持续优化已有 Skill，并通过 FTS5 全文检索跨 Session 召回历史对话。

**核心能力：**
- **自进化闭环**：Task 结束 → 自动提炼 Skill → 下次任务自动加载 → Skill 在使用中再优化
- **多模型无锁定**：支持 Nous Portal / OpenRouter（200+ 模型）/ GLM-Z / Kimi / MiniMax，`hermes model` 一键切换
- **全平台接入**：Telegram、Discord、Slack、WhatsApp、Signal、CLI 共享同一 Gateway 进程
- **分布式运行**：$5 VPS → GPU 集群 → Serverless（Daytona / Modal 按需唤醒）
- **兼容 agentskills.io**：开放 Skill 生态标准，可与其他框架互通
- **OpenClaw 迁移**：`hermes claw migrate` 支持从 OpenClaw 一键迁移（生态竞争白热化）

**安装极简：**
```bash
curl -fsSL https://raw.githubusercontent.com/NousResearch/hermes-agent/main/scripts/install.sh | bash
hermes  # 立即开始对话
```

**后端工程师视角：** "自进化"背后是经典的 RAG + RL 融合——Skill 库是动态 knowledge base，任务执行轨迹是隐式训练数据，这比静态提示词工程有本质区别。值得持续关注其 Skill 质量衰退与分布偏移问题如何解决。

## 二、🔬 ArXiv / HuggingFace 论文精选

### 1. 🔥 HyDRA — 视频世界模型混合记忆架构（cs.CV）

> **论文**：[arXiv:2603.25716](https://arxiv.org/abs/2603.25716) — *Out of Sight but Not Out of Mind: Hybrid Memory for Dynamic Video World Models*
> **HF 上行**：133 个 upvote，今日 HuggingFace Papers 榜首

**核心问题：** 现有视频世界模型把场景当静态画布——动态物体消失出画面后再出现，模型往往把它冻结、变形或直接消失。

**HyDRA 解法：**
- **双角色记忆**：静态背景 Archivist（精确存档）+ 动态物体 Tracker（运动轨迹追踪）
- **时空相关性检索**：只选取相关运动线索 attend，不做全局 attention
- **HM-World 数据集**：59K 高保真视频片段，17 场景、49 种物体、精心设计的出入视野事件

**工程意义：** 机器人导航、自动驾驶、游戏 AI 中的"记住消失的物体"是 safety-critical 能力，这是目前最系统的解法。

### 2. ShotStream — 交互式多镜头流式视频生成（cs.CV）

> **论文**：[arXiv:2603.25746](https://arxiv.org/abs/2603.25746)
> **HF 上行**：110 个 upvote

**核心贡献：**
- 将多镜头生成重新建模为**因果下一镜头预测**（causal next-shot），支持用户中途发新 Prompt 动态指导叙事
- 双 Cache 机制：Global Context Cache（跨镜头视觉一致性）+ Local Context Cache（镜头内时序连贯性）
- Distribution Matching Distillation 将双向教师模型蒸馏为因果学生模型

**影响：** 影视工业 AI 生成流水线、交互式游戏剧情生成的重要基础工作。

### 3. Natural-Language Agent Harnesses — NLAH 论文（cs.CL）

> **论文**：[arXiv:2603.25723](https://arxiv.org/abs/2603.25723)

**核心主张：** Agent 性能的关键瓶颈之一是 harness 设计被埋在控制代码里，难以跨框架复用和科学对比。论文提出 **NLAH**（Natural-Language Agent Harnesses）——把 harness 控制逻辑外化为可编辑的自然语言可执行制品，配合 **IHR**（Intelligent Harness Runtime）执行层。

**意义：** Agent harness 工程化的第一性原理尝试——类比于 OpenAPI 之于 REST API，NLAH 试图给 Agent 控制流提供语言无关的标准描述层。

### 4. Lie to Me — CoT 推理忠实性研究（cs.CL）

> **论文**：[arXiv:2603.22582](https://arxiv.org/abs/2603.22582) — *Lie to Me: How Faithful Is Chain-of-Thought Reasoning in Reasoning Models?*

**核心问题：** 推理模型输出的 Chain-of-Thought 是真实决策过程的反映，还是事后合理化？

**结论摘要：** 研究发现 CoT 与实际决策路径存在显著偏差——模型经常在 CoT 里"说一套、做一套"。这对依赖 CoT 做可解释性审计的系统是重要警示。

**工程影响：** 不要轻信 LLM 的 reasoning trace，特别是在高风险决策场景（医疗、金融、代码安全审计）中。
