### 1. Dicklesworthstone/destructive_command_guard — AI Agent 安全盾牌（Rust，⭐4.3k+）

> **仓库**：https://github.com/Dicklesworthstone/destructive_command_guard
> **语言**：Rust | **协议**：MIT

**这是什么：** 一个高性能的 AI 编码 Agent 安全钩子，在执行前拦截破坏性命令，保护你的代码不被 Agent 意外删除。

**为什么今天火了：** Cursor 0day 漏洞被公开披露（见 HN 218pt），社区对 AI Agent 安全风险的焦虑达到顶点。dcg 在**正确的时间**出现——它是**覆盖 Agent 最多**的安全工具。

**覆盖范围（业界最广）：**
- Claude Code、Codex CLI 0.125.0+、Gemini CLI、GitHub Copilot CLI
- VS Code Copilot Chat、Cursor IDE、Hermes Agent、Grok (xAI)
- Antigravity CLI、OpenCode、Pi、Aider、Continue

**核心能力：**
- **50+ 安全包**：数据库、K8s、Docker、AWS/GCP/Azure、Terraform 等
- **亚毫秒延迟**：SIMD 加速过滤，用户无感知
- **Heredoc/内联脚本扫描**：捕获 `python -c "os.remove(...)"` 和嵌入式 shell 脚本
- **智能上下文检测**：不阻止 `grep "rm -rf"`（数据），但阻止 `rm -rf /`（执行）
- **Fail-Open 设计**：超时或解析错误时不会阻塞工作流
- **Scan Mode**：CI 集成和 pre-commit 钩子，在代码审查中捕获危险命令

**快速示例：**
```
# AI Agent 尝试执行：
$ git reset --hard HEAD~5

# dcg 拦截并阻止：
═══════════════════════════════════════
BLOCKED  dcg
───────────────────────────────────────
Reason: git reset --hard destroys uncommitted changes
Command: git reset --hard HEAD~5
Tip: Consider using 'git stash' first
═══════════════════════════════════════
```

**后端开发者关注点：** 如果你正在使用 Claude Code、Codex 或任何 AI Agent 写代码，这是今天**最应该安装**的工具。一行命令安装，零配置保护。在 Agent 失控成本越来越高的情况下，dcg 是从"好玩"到"可生产"的必备基础设施。

### 2. mereyabdenbekuly-ctrl/clodex-ide — 本地优先、零信任 Agentic IDE（TypeScript，⭐801）

> **仓库**：https://github.com/mereyabdenbekuly-ctrl/clodex-ide
> **语言**：TypeScript/Node | **协议**：AGPL-3.0 | **官网**：https://ide.clodex.xyz

**这是什么：** 一个开源的 Agent 开发环境，核心理念是："模型输出是不可信的输入。权威来自显式策略、隔离运行时和用户控制审查。"

**核心架构原则：**
- **持久化任务**：工程工作建模为具有自身状态、工作区、进程、权限和审查生命周期的持久任务
- **跨领域操作**：文件、Git、终端、浏览器标签、MCP 工具、Runner 统一管理
- **模型路由**：在不同模型之间路由工作，不改变周围工作流
- **审批请求**：高影响操作（shell、网络、浏览器、远程）需要人工批准

**为什么重要：** 当前主流 IDE（Cursor、Windsurf）都是云端优先，Clodex 是**本地优先**的替代方案。在数据隐私和 Agent 安全越来越敏感的时代，"零信任"架构可能成为新范式。

**后端开发者关注点：** 如果你担心 AI IDE 泄露代码或执行未授权操作，Clodex 提供了一套治理框架。AGPL-3.0 协议意味着你可以 fork 并定制自己的安全策略。

### 4. littledivy/mimic — 用 Python 调用任何 App（Python，⭐909）

> **仓库**：https://github.com/littledivy/mimic
> **语言**：Python | **协议**：未指定

**这是什么：** 拦截任何应用的 API 流量，然后自动生成 Python 客户端，让你像调用库一样调用它。

**工作流程：**
```
捕获流量 (mitmproxy) → 提取认证 (mimic.Session) → 生成客户端 (Claude 读取端点)
```

**典型用法：**
```python
from hinge_client import Hinge

acc = Hinge()  # 复用你捕获的会话
recs = acc.get_recommendations()
acc.like(subject_id, comment="hi lol")
```

**技术亮点：**
- 无需逆向工程——通过 mitmproxy 捕获真实流量
- 自动提取 bearer token、device id、session id、cookies 等认证信息
- 生成的客户端是标准 Python，可编辑
- 支持多步调用链（先获取 token，再消费 token）
- 三种构建方式：mitm 捕获、cURL 粘贴、手动构造

**后端开发者关注点：** 如果你在集成没有官方 SDK 的第三方 API（或者 API 文档不全），mimic 提供了一条"从用户行为到 SDK"的捷径。对于内部 API 测试和自动化也很有价值。
