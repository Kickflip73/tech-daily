### 1. msitarzewski/agency-agents — AI Agent 人格化专家团队（今日 ⭐+1,468，总 ⭐10.7k）

> **仓库**：https://github.com/msitarzewski/agency-agents
> **语言**：Shell/Markdown | **协议**：MIT

**这是什么：** 一套精心设计的 AI Agent "人格模板"合集——把 Claude Code 变成一整个虚拟公司。每个 Agent 不是泛化的通用 prompt，而是具有独立人格、专业流程和可交付成果的领域专家。

**覆盖的"部门"：**
- **工程部**：前端开发、后端架构师、移动端、AI 工程师、DevOps、快速原型、安全工程师
- **设计部**：UI 设计师、UX 研究员、UX 架构师、品牌守护者、视觉叙事、"趣味注入器"
- **营销部**：增长黑客、内容创作者、Twitter 运营、社区管理
- **战略部**：产品经理、竞品分析师、用户访谈

**每个 Agent 文件包含：**
- 身份与人格特征
- 核心使命与工作流
- 带代码示例的技术交付物
- 成功指标与沟通风格

**使用方式：** `cp -r agency-agents/* ~/.claude/agents/`，然后在 Claude Code 中激活："激活后端架构师模式，帮我设计一个微服务架构"。

**后端开发者关注点：** 这不是一个框架，而是一套**经验沉淀**。后端架构师 Agent 的 prompt 本身就是一份很好的系统设计 checklist——API 设计规范、数据库选型 Trade-off、扩展性策略。即使不用 Claude Code，这些模板也值得一读。

### 2. openai/skills — Codex 官方 Skills 目录（今日 ⭐+948，总 ⭐12.7k）

> **仓库**：https://github.com/openai/skills
> **语言**：Python | **标准**：Agent Skills Open Standard (agentskills.io)

**这是什么：** OpenAI 官方发布的 Codex Skills 目录。Skills 是 Agent 可发现和使用的指令、脚本和资源的文件夹组合——**写一次，到处用**。

**核心概念：**
- **Agent Skills** 是一个开放标准（agentskills.io），不限于 Codex
- **三级目录**：`.system`（自动安装）→ `.curated`（审核通过）→ `.experimental`（实验性）
- **安装方式**：在 Codex 内用 `$skill-installer` 安装，支持按名称或 GitHub URL

**本质洞察：** 这是 OpenAI 在定义 "Agent 的 npm"——一个标准化的能力分发机制。当 Agent 需要执行特定任务时，不再靠堆砌 prompt，而是加载一个结构化的 Skill 包。这和 OpenClaw 的 Skills、Claude Code 的 Plugins 是同一个方向。

**给后端开发者的启示：** Agent Skills 标准正在成型。如果你在做内部 Agent 平台，值得参考 agentskills.io 的标准设计，提前对齐生态。

### 4. karpathy/autoresearch — AI 自主科研实验框架（HN 热门）

> **仓库**：https://github.com/karpathy/autoresearch
> **语言**：Python | **需求**：单张 NVIDIA GPU

**这是什么：** Karpathy 最新作品。给 AI Agent 一个真实的小型 LLM 训练环境，让它**自主实验过夜**——修改代码、训练 5 分钟、检查结果、保留或丢弃、重复。你睡一觉起来，收获一堆实验日志和（希望是）一个更好的模型。

**设计哲学：**
- **三个核心文件**：`prepare.py`（固定，数据准备）、`train.py`（Agent 唯一可修改的文件）、`program.md`（人类编写的 Agent 指令）
- **固定 5 分钟时间预算**：无论 Agent 改了什么（模型大小、batch size、架构），每次实验固定 5 分钟墙钟时间，结果直接可比
- **Metric**：`val_bpb`（验证集 bits per byte），越低越好
- **预期产出**：~12 次实验/小时，睡一觉 ~100 次实验

**Karpathy 的哲学：**
> "你不再像研究员一样修改 Python 文件，而是编写 program.md 来编程你的研究组织。"

**本质洞察：** 这是"AI 做 AI 研究"的最小可行实验。train.py 里什么都可以改——模型架构、优化器、超参数。Agent 唯一的约束是 5 分钟出结果。未来的 program.md 会越来越像一个"研究组织的代码"——多 Agent 协作、自动文献检索、假设生成与验证。

### 5. blackboardsh/electrobun — TypeScript 极速跨平台桌面应用框架（今日 ⭐+610，总 ⭐8.7k）

> **仓库**：https://github.com/blackboardsh/electrobun
> **语言**：TypeScript + Zig | **平台**：macOS / Windows / Linux

**这是什么：** Electron 的下一代替代品。底层用 Bun 执行主进程，Zig 写原生绑定，目标是"完整的桌面应用解决方案"。

**核心优势：**
- **极小体积**：~12MB 自解压包（使用系统 WebView 时，主要是 Bun 运行时）
- **极小更新**：最小 14KB（使用 bsdiff 差分补丁）
- **主进程隔离**：主进程和 WebView 隔离，带类型化 RPC
- **5 分钟起步**：`npx electrobun init` 即可开始

**已有应用：**
- **Audio TTS**：桌面 TTS 应用（Qwen3-TTS 语音设计/克隆/生成）
- **Co(lab)**：混合浏览器 + 代码编辑器
- **DOOM**：在 Electrobun 上跑 DOOM（两种实现：C 版 + 纯 TS 版）

**对比 Electron/Tauri：** Electrobun 的定位介于两者之间——比 Electron 轻量很多（12MB vs 150MB+），比 Tauri 更 TypeScript-native（主进程直接跑 Bun 而非 Rust）。

## 三、🔍 趋势洞察与深度分析

### 趋势 1：Agent 能力的"模块化分发"正在标准化

今天三个项目从不同角度定义了同一个趋势——**Agent 能力的标准化封装和分发**：

1. **openai/skills**（Codex Skills 目录）— 定义了 agentskills.io 开放标准，Skills 是可复用的指令 + 脚本 + 资源包
2. **agency-agents**（Agent 人格模板库）— 验证了"Agent = 人格 + 流程 + 工具"的封装模式
3. **learn-claude-code**（从零实现 Agent）— "Bash is all you need"揭示了 Agent 的本质是**工具编排 + 上下文管理**

**给后端开发者的启示：** Agent 生态正在经历类似"早期 npm/Maven"的阶段。如果你在做 Agent 平台，现在就应该考虑：你的 Agent 能力如何被打包、分发、复用？agentskills.io 值得跟进。

### 趋势 2：AI 代码追踪与验证成为刚需

今天两个信号同时出现：

1. **git-ai**（Git 扩展追踪 AI 代码）— 每次 commit 自动标注人类/AI 代码占比，`git-ai blame` 显示每行代码的模型和会话
2. **HN："AI 验证债务"文章**（72 分，72 评论）— AI 生成的代码越多，验证成本越高

**深层含义：** 当一个项目 92% 的代码是 AI 写的（git-ai 的真实示例），传统 Code Review 流程需要根本性的重设计。未来的 Review 不是逐行看代码，而是：
- 这段代码的"意图链"是什么？（transcript → requirement → code）
- AI 在哪些地方做了"创造性"选择？
- 哪些部分需要人类验证而不只是 Agent 自测？

### 趋势 3：Agent 并行工作流的工具链成熟

三个工具同时上榜，描绘了"一个人管理 10 个 Agent 并行开发"的完整工具链：

1. **worktrunk**（Git worktree 管理）— 让每个 Agent 有独立的工作目录和分支，`wt switch -c -x claude feat` 一命创建
2. **ralph-orchestrator**（循环编排）— Agent 持续迭代直到任务完成，内置 backpressure（测试/lint 不过就拒绝）
3. **hapi**（远程审批）— 离开电脑也能用手机一键审批 Agent 的权限请求

**给后端开发者的启示：** 如果你还在"一个 Terminal 跑一个 Agent"，是时候升级了。worktrunk + ralph-orchestrator 的组合可以让你同时推进 5-10 个 feature branch，每个由独立 Agent 负责。

### 趋势 4："文件即协议"成为 Agent 交互的新范式

HN 热文"文件是人类和 Agent 交互的界面"（167 分）和今天的多个项目共同验证了一个趋势：

- **program.md**（autoresearch）— 编程 Agent 的行为
- **AGENTS.md / MEMORY.md**（OpenClaw 系列）— Agent 的身份与记忆
- **agency-agents**（人格模板）— 用 Markdown 定义 Agent 专业能力
- **Skills 目录**（openai/skills）— Skill = 文件夹

Markdown 文件正在成为 Agent 时代的"配置文件 + API 协议 + 知识库"三合一。这不是偶然——文件是人类可读、可编辑、可版本控制的最简单形式。
