### 1. Lum1104/Understand-Anything — 代码知识图谱引擎（今日 ⭐+3,999，总 ⭐25,674）

> **仓库**：https://github.com/Lum1104/Understand-Anything
> **语言**：TypeScript

**这是什么：** 将任意代码库转化为**交互式知识图谱**的工具。你可以探索、搜索、甚至用自然语言提问关于代码库的问题。支持 Claude Code、Codex、Cursor、Copilot、Gemini CLI 等主流 AI 编程助手。

**核心能力：**
- 自动解析代码库，构建可交互的知识图谱
- 支持搜索和自然语言问答
- 与主流 Coding Agent 深度集成
- "Graphs that teach > graphs that impress" — 强调教育性和实用性而非炫技

**后端开发者关注点：** 这是代码理解/Code Intelligence 赛道的爆发点。与 codegraph（今日 +3,003⭐）形成直接竞争，两者都瞄准 "让 AI 更高效理解代码库" 这个痛点。区别在于 Understand-Anything 侧重交互式可视化，codegraph 侧重预索引+本地化。

### 4. colbymchenry/codegraph — 预索引代码知识图谱（今日 ⭐+3,003，总 ⭐21,897）

> **仓库**：https://github.com/colbymchenry/codegraph
> **语言**：TypeScript

**这是什么：** 面向 Claude Code / Codex / Cursor / OpenCode / Hermes Agent 的**预索引代码知识图谱**。核心理念："fewer tokens, fewer tool calls, 100% local"。

**技术特点：**
- 预索引模式：提前构建代码的知识图谱表示
- 本地运行：无需云端服务，隐私友好
- 减少 Token 消耗：通过结构化知识减少上下文窗口占用
- 减少 Tool Calls：Agent 不需要频繁调用 grep/read 等工具

**与 Understand-Anything 的对比：** codegraph 更注重"效率"（少 token、少调用、本地化），Understand-Anything 更注重"交互体验"（可视化图谱、自然语言问答）。两者互补大于竞争。

**本质洞察：** AI 编程竞争已从"谁的模型更强"进入**"谁的 Agent 生态更丰富"**阶段。Skills/Plugins 成为新的竞争护城河——类似于智能手机时代的 App Store 战争。微软（dotnet/skills）、Anthropic（官方插件目录）、开源社区同时发力，说明这个赛道已成各方必争之地。

**给后端开发者的启示：**
1. 如果你在做内部工具，考虑为 AI Agent 编写 Skills/插件
2. codegraph 和 Understand-Anything 的思路可以应用到任何需要 AI 理解复杂代码库的场景
3. "预索引 + 本地执行"是降低 AI 编程成本的关键路径

### 趋势 2：AI 安全从理论走向实战攻防

Pwn2Own Berlin 2026 上 **LiteLLM 被攻破**、**AI 平台成为独立目标类别**，加上 Anthropic-Cybersecurity-Skills 的爆火（日增 930⭐），三重信号指向同一个方向：

> **AI 基础设施的安全防护已经刻不容缓。**

攻击面包括：LLM Gateway（如 LiteLLM）、Prompt Injection、Agent Tool 注入、模型窃取。防御侧则需要 ATT&CK 映射、运行时 Guardrails（如 Forge）、安全技能注入。

### 趋势 3：SpaceX IPO → 太空计算新范式

SpaceX 以 $1.75-2T 估值冲刺 IPO，其战略叙事已超出"火箭公司"范畴：
- **Starlink** = 太空互联网基础设施
- **Starship** = 大规模轨道/深空运输
- **太空数据中心** = 突破地面算力的能源/散热瓶颈

这可能是未来 10 年最具颠覆性的基础设施投资主题。

### 趋势 4：中国 AI 应用量领跑，但"量"≠"质"

中国 AI Token 调用量达美国 2.11 倍的数据令人瞩目，但需冷静看待：
- **优势**：应用场景丰富、用户基数大、企业数字化转型快
- **挑战**：高端芯片受限、顶尖模型能力仍有差距、开源生态依赖性强
- **关键观察指标**：算力自给率、前沿论文产出、开源框架国际影响力
