### 1. Yeachan-Heo/oh-my-codex (OmX) — Codex CLI 增强运行时（今日 ⭐+1,789，总 ⭐15,605）

> **仓库**：https://github.com/Yeachan-Heo/oh-my-codex
> **语言**：TypeScript | **协议**：MIT | **NPM**：`oh-my-codex`

**这是什么：** OmX（Oh My codeX）是为 OpenAI Codex CLI 设计的工作流增强层。它不替换 Codex——而是在 Codex 之上叠加一套标准化工作流、Agent 角色体系和持久化状态管理。当日增 1,789 stars，位居今日 TypeScript Trending 第一。

**核心能力架构：**
- **四大内置指令**：
  - `$deep-interview`：需求深度澄清（先问清楚再动手）
  - `$ralplan`：评审 trade-off 后锁定执行计划
  - `$ralph`：持久完成循环（一直跑到任务完成）
  - `$team N:role`：N 个 Agent 并行协作执行
- **启动方式**：`omx --madmax --high`（madmax = 最强提示词模式，high = 高质量输出）
- **状态持久化**：计划、日志、记忆、模式追踪全部保存在 `.omx/` 目录
- **多平台支持**：macOS/Linux（tmux 支持 durable team runtime）+ Windows（psmux）
- **OpenClaw 集成**：项目专门提供了 OpenClaw 集成指南

**技术洞察：** OmX 的核心思路是"把 Agent 的工作流标准化"。与其让每个开发者自己摸索"先澄清再规划再执行"的流程，不如把这套流程固化为语言指令。`$deep-interview → $ralplan → $ralph/team` 是一套可复用的工程化交付范式。

**后端开发者关注点：** 如果你在用 Codex CLI 做代码重构、API 设计或复杂功能开发，OmX 的 `$team 3:executor` 模式可以让 3 个 Agent 并行处理不同模块——不再需要手动 context switching。

### 6. github/copilot-sdk — Copilot Agent SDK 开放公测（今日 ⭐+36，总 ⭐8,206）

> **仓库**：https://github.com/github/copilot-sdk
> **语言**：TypeScript / Python / Go / .NET / Java（Java WIP）

**这是什么：** GitHub 开放 Copilot Agent 引擎的编程 SDK，Public Preview。核心诉求：把 Copilot CLI 背后的 Agent 运行时变成可编程的 API。

**SDK 多语言支持矩阵：**
| SDK | 安装 |
|-----|------|
| TypeScript/Node.js | `npm install @github/copilot-sdk` |
| Python | `pip install github-copilot-sdk` |
| Go | `go get github.com/github/copilot-sdk/go` |
| .NET | `dotnet add package GitHub.Copilot.SDK` |
| Java | WIP（Maven/Gradle） |

**技术架构：** SDK 通过 JSON-RPC 与 Copilot CLI 服务进程通信。SDK 自动管理 CLI 进程生命周期，开发者只需定义 Agent 行为——规划、工具调用、文件编辑等由 Copilot 运行时处理。

**战略意义：** GitHub 正在将 Copilot 从"IDE 插件"升级为"可嵌入任意应用的 Agent 引擎"。这与 OpenAI 的 Responses API、Anthropic 的 Claude API 形成三方竞争，争夺的是企业 AI Agent 基础设施市场。
