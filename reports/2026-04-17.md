### 1. claude-mut — Claude Code 会话记忆插件（今日 ⭐+1,897，总 ⭐59,662）

> **仓库**：https://github.com/inteotmack/claude-mut
> **语言**：TypeScript | **协议**：MIT

**这是什么：** 一个 Claude Code 插件，自动捕获 Claude 在编码会话中做的所有事情（代码修改、文件操作、命令执行），用 Claude 自己的 agent-sdk 压缩成结构化摘要，并注入到未来会话的上下文中。

**核心机制：**
- **全量捕获**：每次编码会话的 Claude 行为轨迹自动记录
- **AI 压缩**：用 Claude 自己压缩成可复用的上下文摘要（不依赖外部服务）
- **上下文注入**：新会话启动时自动注入相关摘要，解决长上下文丢失问题

**本质洞察：** 这是对"Agent 记忆"问题的终端级解法——不依赖框架层，直接在 Claude Code 层面拦截和压缩 Agent 行为历史。和 ReMe 不同，它聚焦于单工具的会话记忆，而不是跨 Agent 的通用记忆系统。

**后端开发者关注点：** 如果你频繁使用 Claude Code 做复杂重构或多轮迭代，这个插件能显著减少"上次做了什么"的 context switching 成本。

### 2. forrestchang/andrej-karpathy-skills — 单文件 CLAUDE.md 编码规范（今日 ⭐+1,150）

> **仓库**：https://github.com/forrestchang/andrej-karpathy-skills
> **语言**：无（纯 Markdown 规范文件）

**这是什么：** 基于 Andrej Karpathy 关于 LLM 编码陷阱观察的 CLAUDE.md 文件。把 Karpathy 总结的四大原则（Think Before Coding、Simplicity First、Surgical Changes、Goal-Driven Execution）编码进 Agent 的 System Prompt。

**四大原则：**
1. **Think Before Coding**：显式陈述假设、呈现多种解释、遇到不确定时停止询问
2. **Simplicity First**：最小化代码，不做 speculative 功能，"如果 200 行能写成 50 行就重写"
3. **Surgical Changes**：只修改与任务相关的代码，不触碰正交代码
4. **Goal-Driven Execution**：测试先行，用可验证的成功标准驱动执行

**本质洞察：** 这个项目之所以突然爆火，是因为它解决了一个所有人都有切身体会的问题——AI 写代码时"自作主张"。把它塞进 CLAUDE.md 就能约束 Claude Code 的行为，不需要改框架。

**后端开发者关注点：** 直接复制到你的项目根目录 CLAUDE.md，立竿见影。

### 3. lsdefine/GenericAgent — 自进化 Agent（今日 ⭐+872，总 ⭐2,746）

> **仓库**：https://github.com/lsdefine/GenericAgent
> **语言**：Python

**这是什么：** 一个自我进化的 Agent：从 3.3K 行种子代码开始，自动生长技能树，最终实现完整系统控制，同时 Token 消耗降低 6x。

**核心机制：**
- **技能树生长**：Agent 根据任务需求自动扩展技能节点，而非预定义工具集
- **Token 优化**：通过技能树剪枝和压缩，将复杂任务的 Token 消耗降低 6 倍
- **系统控制**：最终达到"full system control"——能自主操控整个运行环境

**本质洞察：** 这是目前最有野心的"自进化 Agent"实现之一。和 EvoMap/evolver（基因进化协议）不同，GenericAgent 的进化是任务驱动的技能树扩展，不是随机变异选择。

**后端开发者关注点：** 如果想研究 Agent 自主进化方向，这个仓库的架构设计值得深读。"6x less token consumption" 这个指标也值得关注。

### 7. EvoMap/evolver — GEP 驱动的自进化引擎（今日 ⭐+812，总 ⭐3,140）

> **仓库**：https://github.com/EvoMap/evolver
> **语言**：JavaScript

**这是什么：** 基于 GEP（Gene Expression Programming，基因表达编程）的 AI Agent 自进化引擎。核心思路：把 Agent 的行为规则编码为"基因"，通过进化算法自动优化。

**核心机制：**
- **GEP 编码**：Agent 规则 → 基因序列，可遗传、可变异
- **进化选择**：根据任务执行成功率自然选择最优"基因"
- **持续优化**：无需人工干预，自动迭代改进

**与 GenericAgent 对比：** GenericAgent 是技能树驱动，Evolver 是基因进化驱动。两条路各有千秋——技能树更可控，基因进化更"自然"。

**andrej-karpathy-skills** 的爆火说明一个趋势：LLM 编码规范正在从"专家知识"变成"开箱即用的配置文件"。

**给后端开发者的启示**：把 CLAUDE.md 加进你的项目。它解决的问题——AI 自作主张、过度复杂化、触碰无关代码——都是真实痛点。

---

### 趋势 3：自进化 Agent 的两种路径之争

今天同时有 **GenericAgent**（技能树进化）和 **Evolver**（基因进化）上榜，两条路在并行探索：

| 维度 | GenericAgent（技能树） | Evolver（基因进化） |
|------|---------------------|-------------------|
| 进化机制 | 任务驱动技能扩展 | GEP 基因变异选择 |
| 可控性 | 高（技能树可见） | 低（黑盒进化） |
| Token 效率 | 6x less | 待验证 |
| 适合场景 | 明确任务域 | 开放探索域 |

**给后端开发者的启示**：两条路都有价值。如果你的 Agent 有明确任务域，技能树路径更可控；如果是探索性任务，基因进化可能更有创造力。
