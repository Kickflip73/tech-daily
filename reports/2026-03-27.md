## 三、🔬 arXiv / HuggingFace Papers 学术前沿

### 📄 1. CUA-Suite：大规模人类标注视频数据集，攻克桌面 Agent 瓶颈
> **arXiv**：https://arxiv.org/abs/2603.24440
> **项目**：https://cua-suite.github.io/

Computer-Use Agent（CUA）的核心瓶颈是高质量连续演示数据的极度匮乏——最大开源数据集 ScaleCUA 仅有 200 万截图（< 20 小时视频）。CUA-Suite 提供：
- **VideoCUA**：10,000 个人类演示任务，覆盖 87 款桌面应用，55 小时连续 30fps 录屏 + 600 万帧 + 运动学鼠标轨迹 + 多层推理标注
- **UI-Vision**：GUI grounding & planning 标准评测 benchmark
- **GroundCUA**：56K 截图 + 360 万 UI 元素标注的 grounding 数据集

**核心发现：** 当前 frontier 模型在专业桌面应用的任务失败率约 60%——说明 CUA 离真正实用还有相当大的距离。

**意义：** 数据集的质量决定 CUA 的上限。连续视频 > 离散截图，这个认知已成为领域共识。

### 📄 2. EVA：端到端视频 Agent 的高效强化学习框架（CVPR 2026）
> **arXiv**：https://arxiv.org/abs/2603.22918
> **代码**：https://github.com/wangruohui/EfficientVideoAgent

长视频理解的核心问题：现有 MLLM 要么处理全部帧（计算爆炸），要么均匀采样（丢失关键片段）。EVA 通过**规划先于感知**解决这一矛盾：

**三阶段训练流水线：**
1. **SFT（监督微调）**：建立基础 reasoning 能力
2. **KTO（Kahneman-Tversky Optimization）**：基于人类判断偏好的对齐
3. **GRPO（广义奖励策略优化）**：强化学习精调，优化 query-driven 效率

**Agent 循环**：迭代 Summary → Plan → Action → Reflection，自主决定"看什么、什么时候看、怎么看"。

**性能：** 在 6 个视频理解 benchmark 上超越通用 MLLM baseline 6-12%，超越先前自适应 Agent 方法 1-3%。

---

### 📄 3. T-MAP：轨迹感知进化搜索攻击 LLM Agent（MCP 安全）
> **arXiv**：https://arxiv.org/abs/2603.22341

现有 LLM 红队测试聚焦于让模型输出有害文本，但 Agent 的漏洞发生在**多步工具执行**过程中——特别是 MCP 生态系统中。T-MAP 提出：

**核心创新：**
- 利用执行轨迹（trajectory）而非单步响应来引导对抗性 prompt 搜索
- 自动生成不仅绕过安全护栏、还能通过真实工具调用完成有害目标的攻击链
- 攻击实现率（ARR）显著超越 baseline，且对 GPT-5.2、Gemini-3-Pro、Qwen3.5、GLM-5 均有效

**给 MCP 开发者的警示：** 工具调用链的安全边界比单次 prompt 的安全边界脆弱得多。T-MAP 提供了一个评估框架，建议 MCP Server 开发者用此类方法进行对抗性测试。

---

### 📄 4. UI-Voyager：从失败轨迹自我进化的移动端 GUI Agent
> **arXiv**：https://arxiv.org/abs/2603.24533

长时域 GUI 任务的两大难题：从失败轨迹学习效率低 + 稀疏奖励下信度分配模糊。UI-Voyager 的两阶段方案：

- **Stage 1 — Rejection Fine-Tuning (RFT)**：数据与模型在全自主循环中共同演进
- **Stage 2 — GRSD（Group Relative Self-Distillation）**：识别 group rollout 中的关键分叉点，从成功轨迹构建稠密步级监督

**意义：** 不依赖人工标注，纯从自身失败经验中提升——是 Agent 自主进化能力的重要步骤。

---

### 📄 5. Self-Distillation 为何有时损害 LLM 推理能力
> **arXiv**：https://arxiv.org/abs/2603.24472

Self-Distillation 作为 LLM 后训练范式越来越流行，但在数学推理任务上发现：它有时会缩短 response 长度同时降低性能。根因分析：

**核心发现：** 性能下降源于**认知语言化的抑制**——当 teacher 模型基于丰富上下文生成答案时，会压制不确定性表达，导致学生模型失去在 OOD 问题上的"表达不确定性并适应"能力。

**实验结果：** 在 Qwen3-8B、DeepSeek-Distill-Qwen-7B、Olmo3-7B 上，性能下降最高达 40%。

**启示：** 不确定性表达（epistemic verbalization）是鲁棒推理的关键，不应被训练掉。

---

### 📄 6. Unsupervised Self-Evolution — 无监督多模态推理自我进化
> **arXiv**：https://arxiv.org/abs/2603.21289

不依赖人工标注答案或外部 reward model，纯无监督提升多模态推理能力。核心：对每个输入采样多条推理轨迹，对其置信度分布进行联合建模，通过自我评判选择最优路径进行强化。

### 趋势 4：开发者基础设施的去中心化冲动

Codeberg 迁移帖（501 分）在 HN 持续爆热，反映出开发者社区对平台依赖性的深层焦虑。这不只是 GitHub vs Codeberg 的选择，而是：
- **数据主权**：你的代码、Issue、PR 历史归谁所有？
