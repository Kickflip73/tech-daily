### 5. Nvidia Greenboost — GPU VRAM 透明扩展到系统内存/NVMe（93 points）

> **来源**：GitLab
> **讨论**：21 comments

**这是什么：** NVIDIA 推出的系统软件层，让 GPU 显存不足时**透明地**使用系统 RAM 或 NVMe SSD 作为扩展 VRAM，无需修改上层代码。

**原理：** 基于 CUDA Unified Memory + 自定义 Page Migration Engine，当 GPU 内存压力过高时，自动将冷数据（不活跃的 Tensor）迁移到系统内存，热数据保留在 GPU。

**性能层级：**
```
GPU HBM (最快) → System RAM (PCIe 5.0: ~64GB/s) → NVMe (Gen5: ~14GB/s)
```

**实际意义：** 8GB/12GB 显卡可以运行 30B-70B 参数模型（慢，但能跑），无需花钱升级硬件。本地部署大模型的成本门槛大幅降低。

### 3. Context Engineering 成为新范式

> **仓库**：https://github.com/coleam00/context-engineering-intro（Python Trending）

**什么是 Context Engineering？**

这是 2026 年 AI 编程领域的核心方法论演进：

```
Vibe Coding < Prompt Engineering < Context Engineering
```

**三种层次对比：**
- **Vibe Coding**：随便说说，看 AI 猜测
- **Prompt Engineering**：精心措辞，聚焦于"怎么问"
- **Context Engineering**：系统工程，提供完整上下文（文档、示例、规则、模式、验证）

**标准模板结构：**
```
CLAUDE.md      ← 项目规则 + 约定
examples/      ← 代码示例（最关键！）
INITIAL.md     ← 功能需求描述
/generate-prp  ← 生成 PRP（Product Requirements Prompt）
/execute-prp   ← 按 PRP 执行实现
```

**核心洞察：** "大多数 Agent 失败不是模型的失败，而是上下文的失败。" 这与 Anthropic 的 Model Card 研究结论一致。
