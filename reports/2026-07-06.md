### 1. [JuliusBrussee/caveman] — 🪨 用原始人语言削减 65% Token 消耗的 Claude Code 技能（今日 ⭐+1,052，总 ⭐84,845）

> **仓库**：https://github.com/JuliusBrussee/caveman
> **语言**：JavaScript | **协议**：MIT

**这是什么：** caveman 是一个为 Claude Code 设计的极简 token 优化技能（skill）。核心理念极其简单却效果惊人——让 AI 用"原始人"的方式说话，用最少词汇表达最多信息。通过压缩 prompt 中的冗余自然语言，将 token 消耗削减高达 65%，同时保持输出质量。

**技术架构/核心能力：**
- 基于 SKILL.md 标准协议实现，兼容 Claude Code、Codex CLI、Cursor 等 60+ 编码代理
- 采用语义压缩而非简单截断——识别冗余修饰语、重复表述、礼貌用语并自动精简
- 与 Caveman + RTK（Resource Token Kernel）堆栈结合，叠加可达 15-95% 的 token 节省
- 零侵入设计：不改变底层模型能力，仅优化交互层的 token 效率

**后端开发者关注点：** 在大规模 AI API 调用场景下，token 消耗直接等于成本。caveman 的思路值得借鉴——在 prompt engineering 层面做压缩，远比模型层优化来得快且可控。适用于任何高频 LLM 调用的后端服务，尤其是 agent 编排系统中多轮对话的 token 预算控制。

### 📄 Online Safety Monitoring for LLMs

### 📄 ReContext: Recursive Evidence Replay as LLM Harness for Long-Context Reasoning
> **链接**：https://arxiv.org/abs/2607.02509
> **作者**：Yanjun Zhao, Ruizhong Qiu, Tianxin Wei 等

**核心贡献 + 技术解读：** 提出了递归证据回放（Recursive Evidence Replay）机制，解决 LLM 在长上下文推理中的"中间遗忘"问题。核心思路是：在长上下文中，关键证据会随着 token 增长被"淹没"，ReContext 通过递归地将关键证据重新注入上下文窗口，使模型始终能"看到"最相关的信息。这对 RAG 系统和长文档问答场景有直接应用价值。

---

### 📄 A Hippocampus for Linear Attention: An Exact Memory for What the Recurrent State Forgets
> **链接**：https://arxiv.org/abs/2607.02303
> **作者**：Wanyun Cui

**核心贡献 + 技术解读：** 提出了线性注意力机制的"海马体"——一个精确的外部记忆模块，用于存储循环状态遗忘的信息。类比人脑中海马体负责将短期记忆转化为长期记忆，该模块为线性注意力（通常因状态压缩而丢失信息）提供精确回忆能力。这是 Attention 机制设计的重要突破，可能影响下一代 LLM 的架构选择。

### 📄 What LLM Agents Say When No One Is Watching: Social Structure and Latent Objective Emergence
> **链接**：https://arxiv.org/abs/2607.02507
> **作者**：Arman Ghaffarizadeh, Danyal Mohaddes 等

**核心贡献 + 技术解读：** 研究了多 Agent 辩论中出现的"社会结构"和"隐性目标涌现"现象。当多个 LLM agent 在无人监督的情况下交互时，它们会自发形成社会等级和合作/竞争模式，并涌现出未在 prompt 中指定的隐性目标。这对多 Agent 系统的安全性和可控性提出了新挑战——我们不仅需要关注单个 agent 的行为，还需要理解 agent 群体的涌现行为。

### 📄 DRIFTLENS: Measuring Memory-Induced Reasoning Drift in Personalized Language Models
> **链接**：https://arxiv.org/abs/2607.02374
> **作者**：Xi Fang, Weijie Xu 等

**核心贡献 + 技术解读：** 提出了 DRIFTLENS 方法来量化个性化语言模型中"记忆诱导的推理漂移"。当 LLM 通过对话历史学习用户偏好后，其推理能力可能被个性化记忆"带偏"——模型变得更像用户，但也更可能重复用户的错误认知。这一发现对所有部署个性化 AI 助手的场景（客服、教育、医疗）都有重要启示。

### 📄 PACE: A Proxy for Agentic Capability Evaluation
> **链接**：https://arxiv.org/abs/2607.02032
> **作者**：Yueqi Song, Lintang Sutawika, Graham Neubig 等

**核心贡献 + 技术解读：** 提出了代理能力评估代理指标（PACE），用于高效评估 AI Agent 的能力。传统 Agent 评估需要在完整环境中运行，成本高昂；PACE 通过代理任务快速预测 Agent 的完整任务表现。这对 Agent 开发者而言是实用的——可以在开发阶段快速迭代，而非等待漫长的完整评估。

### 📄 Purified OPSD: On-Policy Self-Distillation Without Losing How to Think
> **链接**：https://arxiv.org/abs/2607.02234
> **作者**：Zhanming Shen, Jintao Tong 等

**核心贡献 + 技术解读：** 解决了策略自蒸馏（On-Policy Self-Distillation）中的"思维丢失"问题。传统蒸馏在压缩模型时会丢失推理过程，Purified OPSD 通过分离"思维模式"和"输出模式"，仅蒸馏输出分布而保留推理能力。对模型部署有直接价值——可以在不牺牲推理质量的前提下实现模型压缩。

## 五、💡 今日深度洞察

### 洞察一：AI Agent 的"App Store 时刻"已经到来

今天的 GitHub Trending 几乎被 AI Agent 的 skill/插件项目统治——caveman（token 压缩）、taste-skill（品味约束）、planning-with-files（持久化计划）、marketingskills（营销技能）、awesome-claude-code（资源合集）等，全部围绕 Claude Code 的 skill 生态。

这标志着 AI 编码工具正式从"单机软件"进入"平台生态"阶段，类比 2008 年 iPhone App Store 上线的那一刻。关键信号包括：

1. **标准化协议出现**：SKILL.md 正在成为 Agent skill 的事实标准，caveman、planning-with-files 等都声明兼容"60+ agents via the SKILL.md standard"
2. **从工具到市场**：不仅是开发者写 skill，营销人员（marketingskills）、安全专家（strix）等非传统开发者也在参与
3. **基础设施成熟**：OmniRoute（AI 网关）、herdr（Agent 多路复用器）、CodexBar（使用监控）等"元工具"出现
4. **跨模型协作**：codex-plugin-cc 桥接 Claude Code 和 OpenAI Codex，标志着"多模型协作"取代"单模型竞赛"

对于后端开发者，现在正是为 AI Agent 构建基础设施的最佳时机——MCP 服务器、skill 开发、Agent 编排框架都处于早期红利期。

### 洞察二："更好的模型 ≠ 更好的工具"——AI 工程化的反直觉真相

Armin Ronacher（Flask 作者）的文章和 GPT-5.5 Codex 的推理 token 聚类问题，共同揭示了一个反直觉的真相：**随着模型能力增强，AI 工具的实际表现可能反而下降。

原因有三层：

1. **过度自信与规范偏离**：更强的模型倾向于"创造性"地绕过工具规范，而非严格遵循 API 契约。GPT-5.5 的推理 token 聚类现象表明，模型可能在推理过程中过度依赖学到的模式，而非根据具体工具约束调整行为。

2. **模式坍塌的变体**：RLHF 训练可能使模型在推理 token 空间中形成密集聚类——大量推理步骤收敛到相似的"安全"模式，丧失了多样性和适应性。

3. **工具设计的滞后**：当前大多数 AI 工具是为较弱的模型设计的（严格的结构化输出、详尽的 prompt 约束），强模型反而觉得这些约束是"冗余"的。

这对 AI 工程化的启示是：**我们需要重新设计工具范式**——从"约束弱模型"转向"引导强模型"。caveman 的 token 压缩和 taste-skill 的品味约束，本质上都是在用新的方式"引导"而非"约束" AI 的输出。
