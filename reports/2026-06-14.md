### 1. obra/superpowers — AI 编程智能体开发方法论（总 ⭐220k，日增 ⭐+920）

> **仓库**：https://github.com/obra/superpowers
> **语言**：Shell/Markdown | **作者**：Jesse Tane（obra）

**这是什么：** 2026 年开发者社区最热的方法论框架。核心论点是：AI Agent 的输出质量，不取决于模型多聪明，而取决于你给 Agent 搭建了什么样的"超能力环境"——上下文管理、架构约束、熵管理（防止规则文件自身腐化）。

**震撼数据验证：** LangChain 的编码 Agent 仅优化 Superpowers 环境，不改底层模型，Terminal Bench 2.0 排名从第 30 名飙升至第 5 名。这不是模型突破，这是**工程范式**的突破。

**五大核心超能力：**
1. **Context Engineering**：精确控制 Agent 在任何时刻"知道什么、不知道什么"
2. **Architecture Constraints**：在 Agent 开始工作前写死约束，防止"自由发挥"
3. **Entropy Management**：持续监控 AGENTS.md 等规则文件是否"腐化"（前后矛盾）
4. **Tool Ergonomics**：精心设计工具接口，让 Agent 不会误用
5. **Memory Scaffolding**：结构化记忆系统，而非让 Agent 自由管理上下文

**后端开发者关注点：** 如果你的团队正在推 AI 辅助开发，这是当前最值得读的方法论文档。它解释了为什么有些团队用 Claude Code 效率翻倍，而有些团队还在反复返工。
