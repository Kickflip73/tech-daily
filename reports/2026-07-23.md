### 2. diegosouzapw/OmniRoute — 免费 AI Gateway 聚合器（今日 ⭐+1,651）

> **仓库**：https://github.com/diegosouzapw/OmniRoute
> **官网**：https://omniroute.online
> **语言**：TypeScript/Node.js | **协议**：MIT

**这是什么：** 免费开源 AI 网关，把 250+ 个 AI 提供商（90+ 免费层）聚合为一个统一 API Endpoint。让 Claude Code、Cursor、Cline 等工具免费用上 Claude / GPT / Gemini。

**核心能力：**
- **250+ AI 提供商**：包括 Kimi、Claude、GPT、OpenAI、Gemini 等 500+ 模型

### 4. ruvnet/RuView — WiFi 穿墙感知系统（今日 ⭐+741）

> **仓库**：https://github.com/ruvnet/RuView
> **语言**：C++/Python | **协议**：MIT

**这是什么：** 用普通 WiFi 信号实现穿墙人体感知。通过 ESP32 采集 Channel State Information (CSI)，检测人员存在、呼吸心率、活动识别——不需要摄像头或穿戴设备。

**技术细节：**
- **CSI 信号处理**：利用 WiFi 多径传播，人体扰动会在 CSI 中留下可测量痕迹
- **ESP32 边缘节点**：$9/节点的低成本硬件
- **脉冲神经网络**：本地自适应学习，30 秒内适应环境
- **多频扫描**：跨 6 个 WiFi 频道扫描，用邻居路由器作为免费雷达
- **加密证明**：Ed25519 见证链确保测量可信
- **多平台集成**：Home Assistant、Apple Home、Google Home、Alexa（Matter Bridge）

**可感知内容：**
- 人员存在和占用计数
- 呼吸率和心率（非接触式）
- 活动识别：行走、坐下、手势、跌倒
- 睡眠监测：睡眠阶段分类、呼吸暂停筛查

**后端开发者关注点：**
- 物联网 + 边缘 AI 的典范项目，$9 ESP32 实现毫米波雷达级功能
- 全本地运行，无云依赖，隐私友好
- Matter 协议集成思路值得智能家居开发者学习

### 6. ayghri/i-have-adhd — AI 输出格式化 Skill（今日 ⭐+1,699）

> **仓库**：https://github.com/ayghri/i-have-adhd
> **语言**：Markdown (Skill) | **协议**：MIT

**这是什么：** 一个给编码 Agent 的 Skill，让它不再"先铺垫再回答"，而是直接给出行动步骤。基于 ADHD 友好的信息组织原则。

**10 条规则：**
1. 以下一步行动开头
2. 多步骤任务编号
3. 每次回复以具体下一步结束
4. 抑制跑题
5. 每轮重述当前状态
6. 具体时间估计（分钟，不是"一会儿"）
7. 让成果可见
8. 错误陈述事实，不铺垫
9. 列表不超过 5 项
10. 无开场白、无总结、无结束语

**效果对比：**
- **Before**："Great question! Let me think about this..." → 300 字铺垫
- **After**："Run `npm install jsonwebtoken@latest`, then edit `src/auth.ts:42`." → 直接行动

**后端开发者关注点：**
- 如果你被 AI 的冗长回复困扰，这个 Skill 模板值得 fork 定制
- 支持 Claude Code、Codex 插件市场一键安装
- 团队可以统一输出规范，提高沟通效率

## 四、🔍 趋势洞察与深度分析

### 趋势 1：AI Gateway / 路由层正在标准化

今天最大的信号是 **OmniRoute 的爆发**（+1,651⭐）。这揭示了 AI 应用层的一个关键趋势：

- **code-review-graph**：AI 工具调用时的 token 优化（38-528x 节省）

**本质洞察：** AI 应用正从"用哪个模型"转向"如何高效、低成本、可靠地路由到正确的模型"。Gateway 层正在从临时脚本走向标准化基础设施。

**给后端开发者的启示：** 如果你在做 AI 应用，尽早抽象出一个模型路由层。未来 6-12 个月，这会成为和 API Gateway 一样的基础设施。

### 趋势 2：AI 编码工具的"周边生态"爆发

今天 GitHub Trending 几乎被 AI 编码工具的辅助项目占领：

1. **i-have-adhd** — Agent 输出格式化 Skill

4. **OmniRoute** — 免费 AI Gateway（面向编码工具）

**本质洞察：** 当 Claude Code、Cursor、Codex 成为主流开发工具后，围绕它们的 Skill 生态、效率工具、成本优化方案正在快速涌现。这是典型的"平台级工具带动周边生态"的 pattern。

### 趋势 3：边缘 AI + 物理感知进入消费级

**RuView** 用 $9 的 ESP32 + WiFi CSI 实现穿墙人体感知，是今天最硬核的项目：

- 无摄像头、无穿戴、无云依赖
- 脉冲神经网络本地自适应
- Matter 协议集成，兼容所有主流智能家居平台
- 加密证明链确保数据可信

**给后端开发者的启示：** 边缘 AI 的成本已经低到可以大规模部署。物联网 + 物理感知 + 隐私保护的组合，正在创造全新的应用场景。

### 趋势 4：语音 I/O 全栈本地化的拐点

**Voicebox** 代表了一个重要趋势：ElevenLabs 和 WisprFlow 这样的云端语音服务正在被开源本地方案替代：

- 7 个 TTS 引擎 + 23 种语言
- 零样本语音克隆
- MCP Agent 语音输出集成
- 完整隐私（数据不离开本机）

**本质洞察：** 语音 I/O 正在从"云服务"变成"本地基础设施"，就像图像生成从 DALL-E 走向 Stable Diffusion 本地部署一样。

### 趋势 5：架构即代码（Architecture as Code）的回归

**LikeC4** 上榜说明开发者对"代码化架构图"的需求在增长：

- C4 模型启发，但更加灵活
- 自定义元素类型、嵌套层级、符号
- VSCode 插件 + 实时预览
- 从代码生成永远最新的架构图

**给后端开发者的启示：** 技术文档和架构图如果不跟代码同步，就会过时。Architecture as Code 是解决这个问题的方法论。
