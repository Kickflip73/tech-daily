### 4. danveloper/flash-moe — 笔记本运行 397B 参数模型（今日 ⭐+290，HN 热榜）

> **仓库**：https://github.com/danveloper/flash-moe
> **语言**：C / Objective-C / Metal

**这是什么：** 纯 C + Metal 实现的推理引擎，在 MacBook Pro (M3 Max, 48GB RAM) 上以 **4.4+ tokens/sec** 运行 Qwen3.5-397B-A17B（397B 参数 MoE 模型）。**无 Python，无框架**。随项目附有完整技术论文（90+ 实验）。

**关键技术突破：**

| 技术 | 说明 |
|------|------|
| **SSD Expert Streaming** | 209GB 4-bit 模型存在 NVMe SSD 上，每个 token 只加载激活的 K=4 个专家（~6.75MB），依赖 OS 页缓存 |
| **FMA-Optimized Dequant** | 把 4-bit 反量化公式从 `(nibble*scale+bias)*x` 改写为 `fma(nibble, scale*x, bias*x)`，预计算后让 GPU FMA 单元一次完成反量化+乘法，提速 **12%** |
| **Apple LLM in a Flash** | 实现了 Apple 2024 年提出的"LLM in a Flash"论文理念 |

**性能矩阵：**

| 配置 | 速度 | 质量 | 备注 |
|------|------|------|------|
| 4-bit experts, FMA kernel | 4.36 tok/s | Excellent | 当前最优，支持 tool calling |
| 2-bit experts | 5.74 tok/s | Good* | 120GB，*JSON 输出破损 |
| 2-bit 单 token 峰值 | 7.05 tok/s | Good* | 不适合 tool use |

**模型架构：** 60 transformer 层（45 GatedDeltaNet 线性注意力 + 15 标准全注意力），每层 512 专家，每 token 激活 K=4 个专家，hidden dim = 4096。

**后端开发者关注点：** 这证明了"大模型本地化"不再只是理论——使用场景包括：离线环境、隐私敏感业务、减少 API 依赖的边缘推理。2-bit vs 4-bit 的 tool calling 可靠性差异是工程决策的重要参考。

### 🔴 The future of version control — Manyana（371 分，218 评论）
> **原文**：https://bramcohen.com/p/manyana
> **HN 讨论**：https://news.ycombinator.com/item?id=47478401

BitTorrent 发明人 Bram Cohen（曾写过原版 BitTorrent、Chia 区块链）发布了新的版本控制系统 **Manyana** 的设计思路文章，HN 社区反应热烈。

**核心痛点（作者的 Git 批评）：**
- Merge conflict 呈现方式混乱：`ours/theirs` 语义含糊，不显示公共祖先（base）
- 历史图可读性差
- Submodule 体验糟糕

**Manyana 的改进：**
- Conflict 标记显示三方：`left / base / right`（而非二方），开发者能清晰看到是谁、在哪个分支改了什么
- 更直观的历史表示
- 专注于解决现实开发流程中的摩擦点

**HN 社区补充：** 多位用户指出 Git 本身通过设置 `merge.conflictStyle = zdiff3` 已能显示 base，所以 Manyana 的差异化需要更多体现在其他层面。

**开发者关注点：** 版本控制是日常工具，Git 已 20 年。在 AI 辅助开发时代（Agent 频繁提交），VCS 的 UX 和 AI-friendliness 重要性上升，Manyana 代表了这个方向的新探索。

## 三、📄 arXiv / HuggingFace Papers 精选

### 🔬 AI / 多模态

**VEGA-3D: Generation Models Know Space**（华科 + 百度，2603.19235）
> **核心**：将视频生成模型中的**隐式 3D 空间先验**复用于场景理解。视频生成模型在训练中必须学习 3D 结构和物理规律才能生成时间一致的视频，VEGA-3D 通过 plug-and-play 框架将这些先验迁移到空间推理任务。
> **意义**：无需显式 3D 数据或几何脚手架，解决了 MLLM 的"空间盲目性"问题。

**Nemotron-Cascade-2（NVIDIA，2603.19220）**
> **核心**：30B 激活参数（A3B）MoE 模型，通过 **Cascade RL + 多领域 On-Policy 蒸馏** 进行后训练。
> **发布**：模型权重和训练数据已开源：https://huggingface.co/collections/nvidia/nemotron-cascade-2
> **意义**：NVIDIA 在 LLM 高效推理 + 强化学习后训练领域持续加码，小激活参数 + 大能力的 MoE 路线。

**SAMA: Factorized Semantic Anchoring and Motion Alignment**（百度 + 清华，2603.19228）
> **核心**：指令引导的视频编辑框架，解决语义修改精确性与运动保真度之间的平衡问题。不依赖外部 VLM 特征或结构条件，内部因子化表示同时建模语义和运动。
> **项目页面**：https://cynthiazxy123.github.io/SAMA

**3DreamBooth: High-Fidelity 3D Subject-Driven Video Generation**（延世大学 + 成均馆大学，2603.18524）
> **核心**：第一个将 3D 主体驱动视频生成的方法。现有方法将主体作为 2D 实体处理，缺乏 3D 几何一致性。3DreamBooth 利用多视角训练数据学习完整 3D 先验，生成视角一致的视频。
> **应用场景**：沉浸式 VR/AR、虚拟制作、下一代电商。

### 🤖 机器人 / VLA（视觉-语言-动作）

**FASTER: Rethinking Real-Time Flow VLAs**（港大 + ACE Robotics，2603.19199）
> **核心问题**：现有 VLA 异步推理方法只优化轨迹平滑度，忽视了**对环境变化的反应延迟**。
> **发现**：反应时间 = TTFA（首次动作时间）+ 执行范围，两者联合决定均匀分布。Flow-based VLA 的常数采样步骤表成为瓶颈。
> **解决方案 FASTER**：引入 Horizon-Aware Schedule，将即时反应动作的去噪压缩到**1步**（在 π₀.₅ 和 X-VLA 中减少 10 倍步骤），同时保持长期动作质量。
> **意义**：机器人真实部署中，100ms 级的反应延迟差异直接决定安全性。

**MoToK: Diffusion-based Discrete Motion Tokenizer**（南洋理工 + 香港中文大学，2603.19227）
> **核心**：提出三阶段框架融合扩散模型（擅长运动学控制）和离散 Token 生成器（擅长语义条件化）的优势：感知 → 离散 Token 生成（规划）→ 扩散运动合成。
> **意义**：人体动作生成的新 SOTA 框架，同时满足语义精准和运动自然两个目标。

### 🧠 Agent / 自演化

**Memento-Skills: Let Agents Design Agents**（HuggingFace Papers 热门，2603.18743）
> **核心**：具备**持续自学习能力**的通用 LLM Agent 系统，让 Agent 自主构建、适应、改进任务专用子 Agent。
> **机制**：
> - 可复用 Skills 存储为结构化 Markdown（行为 + 上下文）
> - Read-Write Reflective Learning：读阶段 → 行为可训练的 Skill 路由器选取最相关 Skill；写阶段 → Agent 基于新经验更新/扩展 Skill 库
> - **无需更新 LLM 参数**，所有适应通过外部化 Skills 和 Prompts 实现
