### 3. volcengine/OpenViking — 字节开源的 AI Agent 上下文数据库（今日 ⭐+1,610，总 ⭐10,510）

> **仓库**：https://github.com/volcengine/OpenViking
> **语言**：Python / Rust / Go | **协议**：Apache 2.0

**这是什么：** 字节跳动火山引擎开源的 Agent 专用上下文管理系统，核心创新是用**文件系统范式**代替传统 RAG 的扁平向量存储。

**痛点定位（精准）：**
- 传统 RAG：向量库存什么→模糊语义检索→黑盒召回→开发者不知道为什么返回错误结果
- OpenViking：文件系统目录组织上下文→支持目录定位 + 语义检索双模式→可视化检索轨迹

**三层上下文架构：**
```
L0 — 活跃层（当前会话直接加载，类似 CPU 缓存）
L1 — 近期层（最近 N 轮任务相关，按需加载）
L2 — 归档层（历史记忆，仅语义检索时触发）
```

**核心特性：**
1. **Filesystem Management Paradigm**：Memory / Resources / Skills 统一目录管理，就像管理本地文件
2. **Tiered Context Loading**：三级结构按需加载，实测可降低 60%+ Token 消耗
3. **Directory Recursive Retrieval**：目录树定位 + 向量检索组合，准确率显著高于纯向量方案
4. **Visualized Retrieval Trajectory**：检索路径可视化，RAG 不再是黑盒
5. **Automatic Session Management**：自动压缩对话历史，提取长期记忆，越用越聪明

**快速上手：**
```bash
pip install openviking --upgrade
# 或 Rust CLI 方式
cargo install --git https://github.com/volcengine/OpenViking ov_cli
```

**后端开发者关注点：** 如果你在做 Agent 平台或内部知识库，OpenViking 解决了两个核心工程问题：1) 如何组织 Agent 的"大脑"（记忆 + 工具 + 资料）；2) 检索过程如何可观测、可调试。L0/L1/L2 三级结构的成本控制策略值得深入研究。

### 1. ExeVRM — 视频执行奖励模型：让 Computer-Use Agent 有了"监工"

> **论文**：Video-Based Reward Modeling for Computer-Use Agents
> **arXiv**：https://arxiv.org/abs/2603.10178
> **机构**：USC / UW / MBZUAI / Amazon AGI

**核心贡献：** 解决 Computer-Use Agent（CUA）评估的扩展性问题。当 AI Agent 在桌面/浏览器/手机上执行任务，如何判断它"真的做对了"？

**技术方案：**
- 输入：用户指令 + Agent 执行过程的视频关键帧序列
- 不依赖 Agent 内部推理过程（模型无关）
- **ExeVR-53k 数据集**：53k 高质量的（视频、任务、奖励标注）三元组
- **对抗指令翻译**：合成步骤级负样本，增强奖励模型鉴别力
- **时空 Token 剪枝**：去除视频中同质化区域和持久 Token，保留决定性 UI 变化

**性能数据：**
- ExeVRM 8B 模型：准确率 **84.7%**、召回率 **87.7%**
- 超过 GPT-5.2 和 Gemini-3 Pro（覆盖 Ubuntu / macOS / Windows / Android）
- 提供比 VLM 更精确的**时序归因**（哪一步出错）

**工程意义：** 这是 CUA 规模化部署的基础设施缺口。有了可靠的视频奖励模型，就能用 RL 自动优化 Agent 策略，不再依赖人工逐步标注。

### 2. IndexCache — DeepSeek 稀疏注意力的跨层索引复用加速方案

> **论文**：IndexCache: Accelerating Sparse Attention via Cross-Layer Index Reuse
> **arXiv**：https://arxiv.org/abs/2603.12201

**核心洞察：** DeepSeek 等模型的稀疏注意力机制中，不同层选中的重要 Token 索引高度相似（跨层稳定性）。既然如此，为什么每层都重新计算索引？

**IndexCache 方案：**
- 缓存前层的 Token 选择索引
- 相似层直接复用，跳过重新计算
- 在保持模型精度的前提下，大幅降低稀疏注意力计算开销

**后端开发者关注点：** 对于在自建 GPU 集群上部署 DeepSeek-V3/R2 的团队，这类优化可直接降低推理延迟和算力成本，且不需要重新训练。

**应用价值：** 机器人感知、自动驾驶感知、AR/VR 场景理解。TTT 范式正在成为 2026 年 AI 系统设计的重要方向——在线自适应 vs 静态模型。

### 4. MADQA — 多模态多跳文档推理基准

> **论文**：Strategic Navigation or Stochastic Search? How Agents and Humans Reason Over Document Collections
> **arXiv**：https://arxiv.org/abs/2603.12180
> **数据集**：https://huggingface.co/datasets/OxRML/MADQA

**核心问题：** AI Agent 在跨文档集合进行多跳推理时，是真的在"策略导航"（识别信息结构、规划检索路径），还是在"随机漫步"（反复尝试直到凑出答案）？

**结论（令人警醒）：** 大多数当前 LLM Agent 在复杂文档推理上呈现随机搜索模式，而不是人类的策略导航模式。为解决此问题，论文提供 MADQA 基准和专属排行榜。
