### 1. thedotmack/claude-mem — Claude Code 智能记忆插件（今日 ⭐+2,305，总 ⭐57,783）

> **仓库**：https://github.com/thedotmack/claude-mem
> **语言**：TypeScript | **协议**：MIT

**这是什么：** Claude Code 的持久记忆插件——自动捕获每次编码会话中 Claude 的所有操作，用 AI 压缩后注入到未来会话的上下文中。

**技术架构：**
- **自动捕获**：拦截 Claude Code 的工具调用（读文件、写文件、bash 命令），生成结构化操作日志
- **AI 压缩**：使用 Claude agent-sdk 对操作日志进行语义压缩，去除冗余信息，保留决策路径和上下文关键点
- **上下文注入**：在新会话启动时，检索历史中相关度最高的记忆片段，作为 MEMORY.md 注入
- **增量更新**：每次会话结束后触发异步压缩，不阻塞主流程
- **去重机制**：向量相似度 + BM25 混合召回，避免重复注入同类记忆

**解决的核心痛点：**
Claude Code 每次启动是全新会话，工程师每次都要重新解释"这个项目的背景是什么""上次遇到什么坑""这里为什么要这么写"。claude-mem 让 Claude 能真正"记住"你的项目历史。

**后端开发者关注点：**
- Java 项目同样适用——代码审查记录、架构决策、踩过的坑全部可沉淀
- 对长期维护大型项目（微服务、单体重构）尤其有价值
- 当前 star 突破 5.7 万，说明"AI 编码助手无状态"是工程师普遍痛点

### 3. virattt/ai-hedge-fund — AI 对冲基金智能体团队（今日 ⭐+1,058，总 ⭐55,036）

> **仓库**：https://github.com/virattt/ai-hedge-fund
> **语言**：Python | **协议**：MIT

**这是什么：** 用 AI Agent 模拟一支完整的对冲基金团队。12 位"AI 投资大师"（巴菲特、芒格、达里奥等风格的 Agent）并行分析，6 位分析师汇总，最终输出投资决策。

**Agent 架构：**
- **研究 Agent**：爬取 SEC 文件、财报、新闻，抽取关键指标
- **风格 Agent**：每个投资风格 Agent 有独立 System Prompt，模拟不同决策逻辑
- **风控 Agent**：独立评估每个建议的风险敞口、相关性、最大回撤
- **汇总 Agent**：加权聚合各 Agent 观点，输出最终仓位建议
- **执行 Agent**：模拟下单（目前为模拟，非真实交易）

**本质洞察：** 这是"多 Agent 协作"架构的金融场景落地。与其说是交易工具，不如说是学习多 Agent 系统设计的绝佳范例。

**后端开发者关注点：** 其 Agent 编排模式（并行研究 → 独立判断 → 加权汇总）可直接迁移到：风险评估系统、多维度 Code Review、需求评审 AI 等场景。

### 5. lsdefine/GenericAgent — 自进化 Agent（今日 ⭐+446，总 ⭐1,908）

> **仓库**：https://github.com/lsdefine/GenericAgent
> **语言**：Python

**这是什么：** 一个从 3,300 行种子代码开始、能自主生长技能树、实现系统级全面控制的 Agent 框架。Token 消耗仅为传统方案的 1/6。

**技术核心：**
- **技能树成长机制**：Agent 在执行任务过程中自动将成功经验抽象为可复用技能，写入技能库
- **自我调用优化**：复杂任务会先检索已有技能，能复用则复用，降低推理开销
- **精简执行路径**：摒弃大量中间推理 token，直接从意图到动作（6x token 效率提升）
- **系统级控制**：浏览器自动化、文件系统、命令执行、API 调用全覆盖

**与主流 Agent 框架的差异：** LangChain/AutoGPT 等是"宽框架"，开发者填空；GenericAgent 是"窄 Agent"，它自己长出工具。

### 6. google/magika — AI 驱动文件类型检测（今日 ⭐+768，总 ⭐13,752）

> **仓库**：https://github.com/google/magika
> **语言**：Python | **机构**：Google | **协议**：Apache 2.0

**这是什么：** Google 开源的 AI 文件内容类型检测工具，比传统基于魔数（magic bytes）的 `file` 命令更准确、更快。

**技术细节：**
- **模型架构**：轻量级深度学习模型（< 1MB），基于文件内容特征而非扩展名或魔数判断
- **准确率**：比 libmagic 高 20%+（在混淆/变种文件上差距更大）
- **速度**：CPU 推理 < 5ms，支持批量处理
- **支持类型**：200+ 文件类型（含各类代码、文档、压缩包、二进制）
- **集成方式**：Python 库 + CLI + WASM（浏览器端可用）

**安全场景应用：**
- 反混淆：检测将恶意代码伪装成图片/PDF 的攻击
- 内容审核：上传文件类型真实性验证
- 恶意软件分析：二进制变种识别

**后端开发者关注点：** 如果你的系统要处理用户上传文件，`magika` 应该作为安全层的第一道防线，替代 `file` 命令或单纯检查 MIME type。`pip install magika` 即可使用。
