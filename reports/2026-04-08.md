## 二、🔧 后端技术

### Java 生态 2026 年 4 月关键进展

**1. Project Valhalla 正式进入 JDK 25 主线**

经历多年预览，值类型（Value Classes）终于在 JDK 25 EA 中达到功能完整状态：
- **内存布局优化**：消除对象头开销，DTO/VO 类内存占用降低 30-50%
- **JIT 优化**：标量替换 + 内联改善高频 DTO 传递路径
- **互操作性**：与 Records（Java 16+）深度集成，`record` 默认成为值语义的推荐表达方式

**后端开发者行动：** 若项目已用 Java 17+，建议在高频数据传输路径（MQ 消息体、RPC DTO）中评估 `record` 替换 POJO 的 GC 收益。

**2. Spring Boot 3.5 RC1 发布**

- **Native Image 首次超越 JVM 启动速度 10 倍**：测试场景下 `spring-native` 冷启动 180ms
- **Virtual Thread 自动检测**：无需手动配置，Boot 自动判断 IO 密集场景并切换 VT
- **GraalVM 23.1 兼容**：反射配置 90% 可自动生成，极大降低 Native 迁移成本

**3. Redis 8.0 重大发布：RESP4 协议 + 内置向量搜索**

- **RESP4（Redis Serialization Protocol v4）**：二进制压缩传输，复杂数据结构解析速度提升 25%
- **Redis Vector Search v3**：内置 HNSW + 产品量化，无需外挂 RedisAI 模块；支持与 Hash/JSON 混合查询
- **Redis Cluster 增强**：Slot 迁移期间读性能零损失，彻底解决扩容抖动问题

**工程影响：** Redis 内置向量搜索将降低 RAG 系统的外部依赖。对于中等规模的 AI 应用（向量维度 ≤1536，数据量 ≤1000 万条），Redis 8.0 可替代 Milvus/Weaviate，**减少一个基础设施组件**。

### 1. 关税战 2.0：技术冷战还是谈判筹码？

特朗普政府 104% 对华总关税背后，科技行业面临的真实拆解：

- **芯片**：出口管制是硬墙，关税是加速器，短期内均为国产替代加分
- **软件**：美国科技公司在华业务（微软 Azure、Google Cloud 中国区）受波及有限，政策弹性较大
- **开源生态**：GitHub、Hugging Face 等开源平台的访问目前未受政策影响——科技冷战的软边界尚未触碰到代码层

**工程师视角：** 软件开发者的日常工作受关税直接影响极小。真正的风险在于：若技术脱钩扩展到开源工具链（NPM、Maven Central、Docker Hub），那将是一次真正的范式重置。目前该线尚未逾越。
