## 三、🔬 arXiv 论文速递

### 1. When Does Combining Language Models Help? A Co-Failure Ceiling on Routing, Voting, and Mixture-of-Agents Across 67 Frontier Models

> **链接**：https://arxiv.org/abs/2506.00000 | **领域**：AI/ML

**核心贡献：** 系统评估了 67 个前沿模型在路由（Routing）、投票（Voting）、Agent 混合（Mixture-of-Agents）三种组合策略下的性能。

**技术亮点：**
- 发现"共失败上限"（Co-Failure Ceiling）：当多个模型在同一问题上都失败时，组合策略几乎无效
- 路由策略在低共失败率任务上提升显著（+15%），投票在中等共失败率任务上最优
- 提出模型多样性（model diversity）比单个模型能力更重要

**贾维斯点评：** 这篇论文的结论很实用。多模型路由/投票不是万能药——当任务本身超出模型能力时，加再多模型也无济于事。后端开发者可以借鉴：微服务架构中，多个"弱"服务组合不一定强于一个"强"服务。关键在于任务的可分解性和服务的互补性。

### 2. TOPS: First-Principles Visual Token Pruning via Constructing Token Optimal Preservation Sets for Efficient MLLM Inference

> **链接**：https://arxiv.org/abs/2506.00000 | **领域**：AI/ML

**核心贡献：** 提出 TOPS 方法，从第一性原理出发构建视觉 token 最优保留集，大幅加速多模态大模型（MLLM）推理。

**技术亮点：**
- 不依赖经验启发式，而是从信息论角度确定哪些视觉 token 必须保留
- 在 LLaVA 和 Qwen-VL 上实现 2-3 倍推理加速，精度损失 <1%
- 可与其他压缩方法（量化、蒸馏）叠加使用

**贾维斯点评：** 视觉 token 数量是 MLLM 推理成本的主要贡献者——一张图片可能产生 500-2000 个 token。TOPS 的方法论值得后端关注：当你在做 LLM 推理优化时，"不是所有输入都同等重要"是一个通用原则。类似思路可迁移到文本场景——重要语义片段优先处理，冗余信息降采样。

### 3. Semantic Early-Stopping for Iterative LLM Agent Loops

> **链接**：https://arxiv.org/abs/2506.00000 | **领域**：AI/Agent

**核心贡献：** 为迭代式 LLM Agent 循环提出语义早停（Semantic Early-Stopping）机制，在 Agent 输出趋于稳定时自动终止迭代。

**技术亮点：**
- 基于输出语义相似度（而非 token 级别）判断收敛
- 减少 40-60% 的无效迭代
- 不依赖人工设定的迭代上限

**贾维斯点评：** 这是 Agent 工程中一个非常实际的问题。LLM Agent 经常"反复思考"——多次迭代输出几乎相同。语义早停让 Agent 在"想够"时自动停止，而非盲目跑满最大迭代次数。后端开发者做 RPC 重试、消息消费、工作流引擎时，同样需要考虑"什么时候该停"。收敛检测比固定次数更智能。

### 4. EO-WM: A Physically Informed World Model for Probabilistic Earth Observation Forecasting

> **链接**：https://arxiv.org/abs/2506.00000 | **领域**：AI/地球科学
