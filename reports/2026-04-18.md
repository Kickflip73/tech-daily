### 1. EvoMap/evolver — GEP 驱动的 AI Agent 自我进化引擎（今日 ⭐+737，总 ⭐4.2k）

> **仓库**：https://github.com/EvoMap/evolver
> **语言**：JavaScript | **官网**：evolver.ai

**这是什么：** 基于 GEP（Genome Evolution Protocol）的 AI Agent 自我进化引擎。灵感来自遗传算法——Agent 的"技能树"通过进化生长，而非预加载。

**核心技术：**
- **GEP 协议**：Genome Evolution Protocol，定义了 Agent 技能的基因编码、变异与遗传规则
- **自主演化**：Agent 从零开始，通过与环境交互逐步"长出"新技能
- **浏览器/终端/文件系统/键盘鼠标/屏幕视觉/ADB 设备**全面覆盖——9 种原子工具 + ~100 行 Agent Loop

**后端开发者关注点：** 这是 Agent 能力动态扩展的新范式。传统 Agent 靠预定义 Skill 列表，而 EvoMap 的 Agent 能随任务需求"进化"出新能力。如果 Agent 平台需要更灵活的能力扩展机制，这个思路值得借鉴。

### 2. lsdefine/GenericAgent — 自进化 Agent（今日 ⭐+845，总 ⭐3.6k）

> **仓库**：https://github.com/lsdefine/GenericAgent
> **语言**：Python

**这是什么：** 一个极简的自进化 Agent 框架。核心哲学：**不要预加载技能——让 Agent 演化它们**。

**核心设计：**
- **9 种原子工具** + 约 100 行 Agent Loop，即可授予 LLM 对本地计算机的系统级控制
- **覆盖能力**：浏览器操作、终端控制、文件系统、键盘/鼠标输入、屏幕视觉、ADB 设备
- **技能树生长**：从 3.3K 行种子代码出发，自动演化出完整技能树
- **Token 效率**：相比传统方案消耗减少 6x

**本质洞察：** 两个项目同日爆发（EvoMap + GenericAgent），说明 Agent 自进化方向正在从学术走向开源实践。不再是"给 Agent 写死的 Skill 列表"，而是"让 Agent 自己长出来"。

## 三、🔍 趋势洞察与深度分析

### 趋势 1：Agent 自进化成为新范式

今日 GitHub Trending 前两名（**EvoMap** + **GenericAgent**）同时指向同一方向：**Agent 不应预加载技能，而应自主演化**。

- **EvoMap**：基于 GEP 协议，技能通过基因编码、变异、遗传来生长
- **GenericAgent**：9 个原子工具 + 100 行 Agent Loop，自动从种子代码演化出完整技能树

**本质**：从"给 Agent 写死的 Skill 列表"到"让 Agent 自己长出来"。Token 消耗减少 6x 是核心卖点，解决了 Agent 能力膨胀带来的成本问题。

**给后端开发者的启示**：如果你在维护 Agent 平台，考虑引入"技能演化"机制，而非持续扩充预定义 Skill 列表。

---

### 趋势 2：浏览器自动化全面 MCP 化

**Chrome DevTools MCP**（35.8k ⭐）持续上榜，说明 MCP 生态正在快速覆盖浏览器自动化能力。

MCP 接入 DevTools 意味着：
1. **Agent 可以直接操作浏览器**，无需编写 Selenium/Playwright 脚本
2. **标准化接口**：所有 MCP 兼容 Agent（Claude Code、Codox 等）均可使用
3. **调试友好**：Agent 调试时可以直接读取 DOM、mock 网络请求

**给后端开发者的启示**：如果你的测试自动化还是基于 Selenium 脚本，考虑迁移到 MCP 协议，这是更可持续的方向。

**对行业的影响**：语音克隆门槛大幅降低，内容创作、游戏 NPC、虚拟助手等领域将迎来新一波创新。
