### 1. tinyhumansai/openhuman — 个人 AI 超级智能助手（今日 ⭐+3,329）

> **仓库**：https://github.com/tinyhumansai/openhuman
> **语言**：Rust | **协议**：开源

**这是什么：** 开源的个人 AI 超级智能体，定位是"你的私人 JARVIS"。集成了桌面 UI、桌面吉祥物（会说话、有表情）、118+ 第三方集成（Gmail/Notion/GitHub/Slack/Stripe 等），支持本地优先的 Memory Tree + Obsidian Wiki 知识库。

**核心能力：**
- **UI 优先设计**：从安装到运行只需几步，无需终端操作，有桌面吉祥物（mascot）作为交互界面
- **118+ 第三方 OAuth 集成**：自动拉取数据到 Memory Tree，每 20 分钟刷新一次上下文
- **Memory Tree + Obsidian Wiki**：本地 SQLite 存储 + Markdown 文件，受 Karpathy obsidian-wiki 启发
- **TokenJuice 智能压缩**：所有工具调用结果经过 token 压缩层处理，降低成本和延迟高达 80%
- **原生语音支持**：STT 输入 + ElevenLabs TTS 输出 + 吉祥物唇形同步
- **模型路由**：自动将任务分发到合适的 LLM（推理/快速/视觉），可选 Ollama 本地模型

**后端开发者关注点：**
- 这是目前最完整的"个人 AI Agent 桌面客户端"开源方案
- Token 压缩层的设计思路值得借鉴——在 LLM 调用前对 HTML/长文本做预处理
- Memory Tree 的分层摘要存储模式（≤3k token chunk → 层级树）对 RAG 系统设计有参考价值

### 4. ruvnet/RuView — WiFi 信号实时空间智能（今日 ⭐+1,715）

> **仓库**：https://github.com/ruvnet/RuView
> **语言**：Rust | **协议**：MIT

**这是什么：** 将普通 WiFi 信号转化为实时空间智能——生命体征监测、存在检测、空间感知，**完全不需要摄像头或任何视频像素**。

**技术栈：**
- **硬件层**：ESP32-S3 WiFi CSI（Channel State Information）数据采集
- **推理层**：Rust 实现的高性能 WiFi-DensePose（WiFi 信号 → 人体姿态估计）
- **算法**：Doppler 提取、跨节点融合、DynamicMinCut、RSSI 追踪
- **插件系统**：MCP 协议支持，可接入 Claude 等 AI Agent
- **部署**：Docker 支持，桌面客户端可用

**性能数据：** Doppler 提取速度提升 5.7x

**应用场景：** 隐私友好的老人跌倒检测、安防监控、智能家居存在感知、健康监测（心率/呼吸 via WiFi）

**技术意义：** 这是 WiFi Sensing 从学术界走向开源落地的标志性项目。Rust 的高性能特性在此场景下发挥得淋漓尽致。

### 6. obra/superpowers — Agent 软件开发方法论框架（今日 ⭐+1,780）

> **仓库**：https://github.com/obra/superpowers
> **语言**：Shell | **协议**：MIT | **版本**：v5.1.0

**这是什么：** 一套完整的面向 Coding Agent 的软件开发方法论。不只是 Skills，而是一套"从需求到交付"的完整工作流。

**工作流（核心流程）：**
1. **需求确认**：Agent 不直接写代码，先退一步问"你到底想做什么"
2. **Spec 拆解**：将需求拆成可读的小块让你审批
3. **实现计划**：生成清晰的实现计划（目标受众："热情但品味一般的初级工程师"）
4. **子 Agent 驱动开发**：启动 subagent 并行执行每个工程任务
5. **自动审查**：Agent 自主审查和 Review，可持续自主工作数小时不偏离计划

**核心理念：** True Red/Green TDD + YAGNI + DRY

**支持的 Agent**：Claude Code、Codex CLI、Gemini CLI、OpenCode、Cursor、GitHub Copilot CLI、Factory Droid

**与 mattpocock/skills 对比：** Superpowers 更偏完整方法论和工作流编排；mattpocock/skills 更偏轻量级单点技能。两者互补使用效果更佳。
