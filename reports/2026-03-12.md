### 8. Microsoft/BitNet — 1-bit LLM 推理框架（HN 热榜 ⭐+288）

> **仓库**：https://github.com/microsoft/BitNet
> **语言**：C++/Python | **协议**：MIT

**这是什么：** 微软官方的 1-bit LLM 推理框架，支持在 CPU 上高效运行 100B 参数模型。

**技术亮点：**
- **1.58-bit 量化**：极致的模型压缩
- **CPU 高效推理**：单 CPU 可运行 100B 模型，达到 5-7 tokens/s
- **能效提升**：x86 上提升 2.37x-6.17x，能耗降低 71.9%-82.2%
- **GPU 支持**：最新版本支持 GPU 推理

**支持模型：**
- BitNet-b1.58-2B-4T
- Llama3-8B-1.58-100B-tokens
- Falcon3 Family (1B-10B)

**后端开发者关注点：** 本地大模型部署的重大突破，单 CPU 跑 100B 模型不再是梦。

### 🔴 Temporal: JavaScript 时间库的 9 年长征（472 分，162 评论）
> **原文**：https://bloomberg.github.io/js-blog/post/temporal/
> **HN 讨论**：https://news.ycombinator.com/item?id=47336989

Bloomberg 工程师 Jason Williams 撰文讲述 Temporal 的 9 年开发历程。Temporal 是 JavaScript 的新日期时间 API，旨在解决 Date 对象的诸多痛点：

**核心改进：**
- **不可变性**：所有 Temporal 对象都是不可变的
- **强类型**：Instant、ZonedDateTime、PlainDateTime 等多种类型
- **原生时区支持**：内置 IANA 时区数据库
- **日历支持**：支持非公历日历

**历程回顾：**
- 2017：Moment.js 维护者 Maggie Johnson-Pint 提出提案
- 2018：进入 TC39 Stage 1
- 2024：终于进入 Stage 3，预计 2026 正式发布

**后端开发者关注点：** 终于可以告别 moment.js 了！Temporal 将显著改善 JS 的日期处理体验。

**给后端开发者的启示：** 本地大模型部署的成本正在快速下降，隐私敏感的 AI 应用将迎来春天。

MiroFish 和 BettaFish 两个项目本周同时爆发，显示群体智能正在成为 AI Agent 的下一个热点：

- 多个 Agent 协作解决问题

**给后端开发者的启示：** 多 Agent 协作系统的设计和编排是值得投入的方向。

### 趋势 3：Temporal 即将正式发布

JavaScript 最后一个"大山"——日期时间处理——终于要解决了。Temporal 预计 2026 年正式进入 ES 规范。

**给后端开发者的启示：** 
- 可以开始准备从 moment.js 迁移
