### 1. mvanhorn/last30days-skill — AI 多源研究综合技能（今日 ⭐+1,308，总 ⭐15,284）

> **仓库**：https://github.com/mvanhorn/last30days-skill
> **语言**：Python | **定位**：Claude Code 专用 AI Research Skill

**这是什么：** 一个面向 Claude Code 的 AI Agent Skill，可以跨 Reddit、X（Twitter）、YouTube、HN、Polymarket、Bluesky 等多平台同时检索过去 30 天的讨论数据，自动聚合成带有真实引用的研究简报。今天以 **1,308 stars** 居 GitHub 全站日榜首位。

**核心功能（v2.9.5）：**
- **多源并行爬取**：Reddit（ScrapeCreators）、X、YouTube、HN、Polymarket、Instagram Reels、Bluesky（AT Protocol）7 路并发
- **Comparative Mode**：`/last30 Claude Code vs Codex` 可触发三轮并行研究 + 对比表格 + 数据驱动结论
- **自动归档**：每次运行将完整报告保存至 `~/Documents/Last30Days/<topic>.md`，构建个人研究资料库
- **Per-project .env**：在 `.claude/last30days.env` 存放项目专属 API key
- **SessionStart 验证**：Claude Code 会话启动时自动校验配置完整性

**技术亮点 — 信息质量工程：**
```
数据信号 → 评分去重 → 精选高质量 top comments（+10% 权重）
→ 多模态综合（视频标题 + 文字帖 + 预测市场赔率）
→ 叙事生成（citation-grounded narrative）
```

**为何爆火：** Agent Skill 生态正在形成标准化分发市场（ClawhHub、Agentskills.io），`last30days` 填补了"快速感知某领域最新动态"的高频刚需。对于技术决策者，30 分钟人肉研究 → 30 秒自动完成，ROI 极高。

**后端工程师视角：** 底层是经典的 MapReduce 思路——多源并行 fetch → normalize → dedup → rank → reduce 到最终报告。Polymarket 价格数据引入"社区预测"维度，是比传统搜索更有信号价值的特征。

**应用场景：** 会议转录、播客字幕、语音驱动 AI Agent（输入管道）。配合 Vibing（社区 fork）还可实现系统级语音输入法。

### 🔐 安全：ChatGPT Cloudflare Bot 检测解密（289 points）

**来源**：[ChatGPT won't let you type until Cloudflare reads your React state](https://www.buchodi.com/chatgpt-wont-let-you-type-until-cloudflare-reads-your-react-state-i-decrypted-the-program-that-does-it/)

**事件概述：** 安全研究员逆向分析 ChatGPT 的 Cloudflare Turnstile 集成，发现在用户开始输入之前，**Cloudflare 脚本已经扫描了 React 组件树的状态**——不只是鼠标行为，而是应用层内部状态。

**技术细节（已公开部分）：**
- Cloudflare Turnstile 注入的 JS 代码会通过 React DevTools 全局钩子（`__REACT_DEVTOOLS_GLOBAL_HOOK__`）读取 Fiber 树
- 收集的信息不止传统的"鼠标轨迹/键盘节奏"，包含应用状态快照
- 目的是更精确地区分 bot 和真人用户

**工程师关注点：**
1. **隐私边界模糊**：前端应用状态被第三方安全脚本访问，超出大多数用户预期
2. **React 架构的副作用**：React Fiber 树的可观察性使得 bot 检测可以利用"应用语义"而非纯交互信号
3. **对爬虫工程的影响**：传统 headless 浏览器模拟鼠标已不够，需要同时模拟合法的应用状态转换

**HN 社区热评：** 高赞评论关注这是否构成对 CSP（Content Security Policy）精神的违反——毕竟 Turnstile 是第三方脚本，却在读取第一方应用内部状态。

### ⚡ 编程语言：C++26 标准正式完成！（153 points）

**来源**：[C++26 is done – ISO C++ Trip Report, March 2026](https://herbsutter.com/) — HN 153 points

**C++26 关键特性（已锁定）：**
- **Contracts（契约式编程）**：终于在 C++26 落地！`pre`/`post`/`assert` 正式语言级支持
- **Reflection（静态反射）**：`std::meta`，编译期完整反射能力，是 C++ 史上最大特性之一
- **Pattern Matching 的进一步增强**：`inspect` 语句更完善
- **`std::execution`（异步执行框架）**：P2300 最终入标准，基于 sender/receiver 模型
- **`std::inplace_vector`**：固定容量栈分配 vector
- **`std::optional` 的 monadic 接口扩展**

**Contracts 历史背景：** 这个特性在 C++20 被撤回，C++23 仍未入，C++26 终于完成——历时接近 10 年争论。

**后端工程师关注：** `std::execution` 对高性能异步后端（类似 Java 的 CompletableFuture / Kotlin Coroutines）有重大意义，Contracts 则让接口文档变成可执行的运行时保证。

### 2. PixelREPA：像素空间扩散 Transformer 的表示对齐突破

**论文**：Representation Alignment for Just Image Transformers is not Easier than You Think（KAIST AI）

**关键发现：**
- REPA（表示对齐）在潜在空间 DiT（Diffusion Transformer）上有效，但**直接用于像素空间（JiT）会 *恶化* FID**
- 根因：解噪发生在高维图像空间，而语义目标被强压缩，造成"信息不对称"——模型走捷径

**PixelREPA 解决方案：**
- 变换对齐目标（Target Transformation）
- Masked Transformer Adapter：浅层 Transformer + 部分 Token Masking，防止捷径优化

**结果：**
- JiT-B/16：FID 从 3.66 → 3.17，IS 从 275.1 → 284.6
- PixelREPA-H/16：FID=1.81，IS=317.2（新 SOTA）
- 训练收敛速度提升 >2×
