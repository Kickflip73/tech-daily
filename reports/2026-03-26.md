### 🔴 Google TurboQuant：极致压缩重定义 AI 效率（482 分，129 评论，ICLR 2026）
> **原文**：https://research.google/blog/turboquant-redefining-ai-efficiency-with-extreme-compression/
> **论文**：https://arxiv.org/abs/2504.19874

Google Research 正式开源 TurboQuant，一个**零精度损失的向量量化压缩算法**，将于 ICLR 2026 发表。

**核心技术：**
- **PolarQuant**（AISTATS 2026）：随机旋转数据向量，简化几何结构后量化，无需额外存储量化常数
- **Quantized Johnson-Lindenstrauss（QJL）**：结合经典降维理论，实现极低 bit 的高质量向量压缩
- **TurboQuant = PolarQuant + QJL 的最优组合**：彻底消除传统量化的 1-2 bit 存储开销

**实测效果：**
- KV Cache 压缩至 2-3 bit，**零精度损失**（传统方法在 < 4 bit 时有明显退化）
- 向量相似度搜索速度提升 3-5x
- 内存占用降低 70%+

**后端/ML 开发者关注点：** KV Cache 是大模型推理的核心瓶颈之一（长上下文时尤甚）。TurboQuant 把 2 bit 量化做到零损失，是推理优化领域的重要突破。配合 LMCache 使用，可以在降低 70% 内存的同时维持 TTFT 和精度。

## 三、🔬 arxiv & HuggingFace Papers 精选

### 1. MinerU-Diffusion：用扩散模型重思文档 OCR
> **论文**：https://arxiv.org/abs/2603.22458
> **机构**：上海人工智能实验室 / OpenDataLab / 北京大学

**核心思想：** 传统 OCR 用自回归解码（逐 token 串行生成），天生有序列延迟和错误累积问题。MinerU-Diffusion 把文档解析重新定义为"逆向渲染"问题——既然文档排版是从结构化内容"渲染"出来的，OCR 就是这个过程的逆向。

**方法：** 用扩散模型（并行去噪）替换自回归解码，采用分块扩散解码器 + 不确定性驱动课程学习策略，实现长文档的稳定训练和高效推理。

**结果：** 解码速度提升 **3.2x**，对噪声/复杂布局的鲁棒性显著提升。作者同时发布了 Semantic Shuffle 评测基准，验证模型更少依赖语言先验（真正在"看"而不是"猜"）。

**后端开发者关注点：** 如果你有文档处理 Pipeline（发票、合同、学术论文解析），这套方案比 Tesseract/PaddleOCR + VLM 的组合快 3 倍以上，且在排版复杂文档上准确率更高。

### 2. ThinkJEPA：用推理模型赋能潜在世界模型
> **论文**：https://arxiv.org/abs/2603.22281

**核心思想：** JEPA（Joint Embedding Predictive Architecture，Yann LeCun 的世界模型架构）擅长学习物理世界的预测性表示，但缺乏高层推理能力。ThinkJEPA 把大型视觉语言推理模型（类似 o3）的推理能力蒸馏到 JEPA 的潜在空间中，实现"会思考的世界模型"。

**意义：** 在 LeCun 的 AI 路线图（从 LLM 转向 World Model）上迈出重要一步。JEPA + 推理能力 = 更接近真实世界理解的 AI。

### 3. SpecEyes：多模态 LLM 推理的推测性加速框架
> **论文**：https://arxiv.org/abs/2603.23483
> **机构**：厦门大学 / 某研究机构

**核心问题：** 多模态 LLM（如 o3、Gemini Agentic Vision）的"视觉工具调用循环"引入了严重的串行延迟（感知 → 推理 → 工具调用 → 再感知）。

**方法（SpecEyes）：** Agent 级别的推测性加速框架，用小模型预测大模型的感知结果和工具调用序列，大模型并行验证，失败时回滚。将推测性解码从 token 层面扩展到"感知-规划"层面。

**结果：** 在保持精度的前提下，多轮视觉推理任务延迟降低 2-4x。

---

### 4. LLM Agent 工作流优化综述：从静态模板到动态运行图
> **论文**：https://arxiv.org/abs/2603.22386
> **机构**：RPI + IBM Research

**核心框架：** 把 LLM 工作流统一抽象为 **Agentic Computation Graph（ACG）**，按"结构确定时机"分类：
- **编译期静态图**：固定 DAG，确定性强，适合生产环境
- **运行期动态图**：Agent 边执行边决定下一步，灵活但难调试
- **学习期自适应图**：通过强化学习或进化算法自动优化工作流结构

对当前 LangGraph、AutoGen、TaskWeaver 等主流框架都有覆盖分析。是 Agent 工程领域今年最系统的综述之一。

### 5. WildWorld：1.08 亿帧的世界模型数据集（来自《怪物猎人：荒野》）
> **论文**：https://arxiv.org/abs/2603.23497
> **机构**：Shanda AI Research Tokyo / 清华 / BIT / Stepfun

**核心贡献：** 从 AAA 游戏《怪物猎人：荒野》自动采集了包含 **1.08 亿帧**、**450+ 动作类别**的行动条件世界建模数据集，每帧同步标注角色骨架、世界状态、相机位姿和深度图。

**意义：** 现有世界模型数据集要么动作空间简单（Minecraft 只有 WASD），要么缺乏显式状态标注。WildWorld 填补了"语义丰富动作 + 完整状态标注"的空白，是 AI 游戏 + 机器人学习的重要基础设施。

### 6. PEPO：多模态 CoT 推理的 Token 级策略优化
> **论文**：https://arxiv.org/abs/2603.22847
> **机构**：南开大学 / 快手

**核心问题：** 现有 RLVR（可验证奖励 RL）方法对推理链的所有 token 均等优化，忽视了视觉接地（感知）和探索推理（思考）两类 token 的本质差异。

**方法（PEPO）：** 从隐藏状态相似度推导"感知先验"，结合 token 熵通过平滑门控机制计算 token 级优势值（advantage），在 GRPO/DAPO 框架上即插即用，无需额外监督信号。

**结果：** 几何推理、视觉接地、视觉谜题等多项 Benchmark 上超越强基线，训练稳定性更好。

---

### 7. CanViT：主动视觉基础模型
> **论文**：https://arxiv.org/abs/2603.22570

**核心思想：** 主动视觉（Active Vision）——AI 不是被动接受图像，而是主动决定"看哪里"（移动摄像头/视点）。CanViT 提出了 **Active-Vision Foundation Model（AVFM）** 范式，用 ViT 架构从零设计主动视觉预训练框架，将预训练与最终观察策略解耦。

**亮点：** 训练完成的 CanViT 具备零样本对应关系能力，可直接迁移到机器人（电机摄像头控制）和视频理解任务。代码和 HuggingFace 兼容 Checkpoint 已开源。

### 趋势 2：量化技术迎来关键突破——Google TurboQuant 零损失 2-bit

今日 HN 第二大热点：TurboQuant（482 分）+ 量化入门文章（174 分），共同说明 **KV Cache 量化**正在从研究走向工程实践。

**为什么重要：** 长上下文推理的最大瓶颈是 KV Cache 内存。目前 128K token 的 KV Cache 在 FP16 下约需 8-16GB，极大限制了并发能力。TurboQuant 把它压到 2-3 bit 且**零精度损失**，理论上可以让同等显存支持 5-8x 的并发。

**下游影响：** 这对 LLM 推理服务商（vLLM、SGLang、TGI 等）是直接利好，预计 6 个月内主流推理框架会集成。对后端工程师来说，这意味着自建推理服务的成本门槛将大幅下降。
